从mysql无法启动到crypt欺诈病毒解密

发表于 2016-05-10   |   分类于 WEB开发   |     |   本文阅读量 次

今天维护的一台服务器上网站加载不出数据了，首先想到mysql数据库服务是否启动，打开服务管理器双击mysql服务发现启动按钮是灰色的，通过cmd下直接用命令登录提示连接不上数据库，就想着系统是否有问题，于是打开事件查看器，双击错误打开如下图所示：

看到如上图的确有错误信息，来源MSDTC。那MSDTC是什么呢？是一个windows服务。

我们用“sc qc”命令查询这个服务：

从图中可以看到它的DISPLAYNAME IS Distributed Transaction Coordinator,so we open the windows services,find “Distributed Transaction Coordinator”this service，open it：
通过如上操作，没有获得可以有效的解决方法，最后就看了下mysql/data下的.err日志文件，显示如下错误：

[ERROR] Fatal error: Can’t open and lock privilege tables: Table ‘mysql.user’ doesn’t exist

当时也惶惶的没有仔细看上面这个提示错误，就通过我的电脑->右击管理进入本地用户和组,把mysql用户的密码修改为了西部建站工具中设置的密码(windows下修改mysql密码也弄明白了),然后找到服务里面的mysql服务,点击属性->登录选项卡中也设置为刚才设置的密码，还是有一定效果的，就是启动按钮可以使用了，我点击启用，等了一会却提示1053错误，找了些网上说的方法多数是linux服务器的（初始化数据库就ok，即：mysql_install_db –user=mysql），最后就还是的回到mysql日志来看下具体错误是什么，再次看了下上面的错误提示，就是说mysql下的user表有问题（记录mysql管理员账号密码啥的），我就看了下mysql文件下的user表，发现所有文件都被添加了.crypt后缀，才明白服务器中了CryptoWall4.0勒索病毒，最为郁闷的是把mysql备份的数据库也做了加密，以前太大意了，都是简单的把data目录在当前目录就备份了，也没想到服务器整体中招带来的后果。
