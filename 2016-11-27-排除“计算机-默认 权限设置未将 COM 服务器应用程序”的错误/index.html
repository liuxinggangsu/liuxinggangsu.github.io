排除“计算机-默认 权限设置未将 COM 服务器应用程序”的错误

发表于 2016-11-27   |   分类于 转载日志   |     |   本文阅读量 次


服务器刚装好没多久就出现了意外关闭，网上看了看有说电脑死机啥的导致，我一看cpu很低，就eventvwr一下事件查看器，出现下图密密麻麻的错误很多：

根据提示信息，实际上是说NETWORK SERVICE没权限激活CLSID为{BA126AD1-2166-11D1-B1D0-00805FC1270E}的应用程序。
可以通过使用组件服务管理工具修改此安全权限。

  • 1、如果按上面的提示去使用组件服务管理工具找CLSID为{BA126AD1-2166-11D1-B1D0-00805FC1270E}的应用程序是找不到的。
  • 2、需要先运行regedit.exe在注册表中查找出{BA126AD1-2166-11D1-B1D0-00805FC1270E}对应的AppID值{27AF75ED-20D9-11D1-B1CE-00805FC1270E}

可以找到此DCOM属于Network Connection Manager Class，简称为netman

  • 3、然后再打开组件服务，点击“开始”-“控制面板”-“管理工具”-“组件服务”-“计算机”-“我的电脑”-“DCOM”选项，查看方式为详细信息，找到DCOM 配置里的netman，选中按鼠标右建选属性（却发现没有属性选项，
    在开始点运行:
    输入以下文字:
    msdtc –resetlog 这个是重置日志的，我也没明白为啥重置下日志就有属性选项了）。

  • 4、在netman属性里的安全 -> “启动和激活权限” -> 自定义编辑，在启动权限里加入NETWORK SERVICE用户，允许本地启动和本地激活，确定后就不会再报这个DCOM错了。
